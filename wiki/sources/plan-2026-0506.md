# 内容关联功能设计摘要

原始文件：`docs/plan/plan-2026-0506.md` — Phase 6 内容关联设计规划

## 背景

四大技能内容与跨技能内容之间原本没有显式关联。`skill_tags` 只能标注"适用技能"，无法表达"这道听力题考的就是这个同义替换"。

## `ielts_content_links` 表结构

```
source_type  VARCHAR(30)  -- LISTENING / READING / WRITING / SPEAKING
source_id    UUID
target_type  VARCHAR(30)  -- WORD / PHRASE / PARAPHRASE / PRONUNCIATION / GRAMMAR_POINT
target_id    UUID
link_type    VARCHAR(30)  -- vocabulary / paraphrase / grammar / pronunciation / signal
note         TEXT
UNIQUE(source_type, source_id, target_type, target_id)
INDEX idx_links_source(source_type, source_id)
INDEX idx_links_target(target_type, target_id)
```

**无物理 FK**（多态关联），服务层负责级联清理。

## 关联场景

| 技能内容 | 可关联跨技能内容 | 典型用途 |
|---|---|---|
| 听力 | PARAPHRASE | 核心考点：听到A写B |
| 听力 | PHRASE | 信号词（however/on the other hand）|
| 阅读 | PARAPHRASE | 判断题/匹配题的改写考点 |
| 写作 | GRAMMAR_POINT | 范文涉及的语法结构 |
| 口语 | WORD/PHRASE | 话题高分词汇/口语连接词 |

## 前端交互方案

**选定方案**：独立关联 Offcanvas（不覆盖列表，职责清晰）

技能内容页（听力/阅读/写作/口语）：操作列加「关联🔗」按钮，点击弹出 Offcanvas  
跨技能内容页（词/短语/同义替换/发音/语法）：操作列加「引用↩N」按钮，反向查看被哪些技能内容引用

## linkCount 避免 N+1

跨技能内容 list API 响应中直接附带 `linkCount` 字段，后端在 `findAll` SQL 中内嵌 correlated subquery 统计，不逐行请求。

## study.html 翻卡扩展

`StudyPlanItem` 扩展 `linkedItems: List<ContentLinkDto>`，`getTodayPlan()` 返回时附带关联内容快照（仅摘要）。翻卡背面在答案/解析下方展示配套内容区，前端直接渲染，无需额外请求。
