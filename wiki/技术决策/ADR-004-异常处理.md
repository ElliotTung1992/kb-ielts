# ADR-004 异常处理设计

**状态**: 已决策  
**日期**: 2026-05

## 决策

使用 `@RestControllerAdvice` 统一捕获异常，返回标准 `ApiResponse` 格式。业务代码直接 throw，不在 Service 层包装响应。

## 异常类型

```
RuntimeException
└── KbException                  通用业务异常基类
    ├── ResourceNotFoundException  404 — 实体不存在
    └── ResourceExistException     409 — 实体已存在（唯一键冲突）
```

## Handler 映射

`IeltsExceptionHandler.java`：

| 异常 | HTTP 状态 | 说明 |
|---|---|---|
| `ResourceNotFoundException` | 404 | "资源不存在" |
| `ResourceExistException` | 409 | "资源已存在" |
| `MethodArgumentNotValidException` | 400 | Bean Validation 失败，逐字段返回错误 |
| `KbException` | 500 | 通用业务异常 |

## 响应格式

```json
{
  "success": false,
  "data": null,
  "message": "word already exists: abandon"
}
```

400 校验失败时 `message` 为第一个字段错误信息。

## 约定

- Controller 不写 try-catch
- Service 层抛业务异常时使用 `ResourceNotFoundException` / `ResourceExistException` 而非直接 `KbException`
- 新增异常类型继承 `KbException` 并在 Handler 中注册
