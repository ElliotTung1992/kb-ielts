# ADR-001 技术栈选型

**状态**: 已决策  
**日期**: 2026-05

## 决策

采用 Spring Boot 3.4.1 + MyBatis + PostgreSQL 16 构建单体应用。

## 备选方案

| 方案 | 否决原因 |
|---|---|
| Spring Data JPA / Hibernate | 复杂查询可读性差，学习成本高，自动 DDL 与 Liquibase 冲突 |
| MySQL | PostgreSQL UUID、TIMESTAMPTZ、JSON 支持更好；个人偏好 |
| 微服务拆分 | 过度设计，个人项目单体足够 |
| 前后端分离（Vue/React）| 引入构建链复杂度，静态 HTML 够用 |

## 理由

- **MyBatis**：SQL 完全可见，方便调试；PageHelper 分页零侵入
- **PostgreSQL**：原生 UUID、全文检索潜力、JSONB 扩展性
- **Java 21**：虚拟线程就绪，长期 LTS
- **单体**：个人项目，部署简单，Docker Compose 一键启动

## 影响

- Mapper XML 文件较多（每实体一个），维护成本随实体数线性增长
- 无 JPA 自动建表，Schema 变更必须走 Liquibase changeset
