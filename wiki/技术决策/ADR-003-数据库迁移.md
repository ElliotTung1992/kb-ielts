# ADR-003 数据库迁移策略

**状态**: 已决策  
**日期**: 2026-05

## 决策

使用 Liquibase SQL 格式 changeset 管理 Schema，禁止直接修改已发布的 changeset。

## 规则

1. **只追加**：已执行的 changeset 不得修改，新需求写新文件
2. **文件命名**：`NNN-描述.sql`，序号三位（001、009…）
3. **格式**：Liquibase SQL 格式（`--changeset author:id`），非 YAML
4. **入口**：`db.changelog-master.xml` include 所有 SQL 文件

## 当前 Changeset

| 文件 | 内容 |
|---|---|
| `001-kb-ielts-schema.sql` | 全量主 Schema（所有核心表）|
| `009-create-topic-tags.sql` | 话题标签表 |

## 开发流程

```bash
# 本地验证迁移
mvn liquibase:update

# 查看 SQL 不执行
mvn liquibase:updateSQL
```

## 注意

- `ielts_databasechangelog` 和 `ielts_databasechangeloglock` 是 Liquibase 控制表，不要手动修改
- Docker Compose 启动时 app 会自动执行迁移
