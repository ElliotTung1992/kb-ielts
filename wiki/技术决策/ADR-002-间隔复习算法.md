# ADR-002 间隔复习算法

**状态**: 已决策  
**日期**: 2026-05

## 决策

使用 SM-2 变体实现间隔复习，封装在 `SpacedRepetitionCalculator.java`。

## 算法核心

两种模式，由 `SpacedRepetitionCalculator` 内部的 `SM2_TYPES` 集合控制：

**完整 SM-2（WORD / PHRASE）**

| 评分 | interval | ease_factor | 状态变化 |
|---|---|---|---|
| AGAIN | 重置为 1 | ef - 0.20 | LEARNING |
| HARD | × 1.2 | ef - 0.15 | — |
| GOOD | 标准递增 | — | rep≥5 且 interval≥21 → MASTERED |
| EASY | × 1.3 | ef + 0.15 | — |

**阶段式模式（其他所有内容，2026-05-07 更新）**

| 评分 | interval | 掌握条件 |
|---|---|---|
| AGAIN | 1 天 | — |
| HARD | 1-3 天 | — |
| GOOD | 3→7→14→30+ 天递进 | rep≥4 且 interval≥30 → MASTERED |
| EASY | 7→21→45+ 天递进 | rep≥3 且 interval≥30 → MASTERED |

> [!note] 评分枚举
> 使用 `AGAIN / HARD / GOOD / EASY`，不是数字 0-5。

> [!contradiction] 与 [[sources/sequence-study]] 冲突
> `sequence-study.md`（设计初稿）描述简化模式为 GOOD/EASY 直接 → MASTERED interval=365。
> `plan-2026-0508.md`（2026-05-07 实现）将其改为阶段式，以当前代码为准。

## 存储

SM-2 状态存入 **`ielts_study_records`**（不是 review_logs）：

```sql
ease_factor      DECIMAL  -- 初始 2.50
interval_days    INT      -- 当前间隔天数
repetition_count INT      -- 连续成功次数
next_review_at   DATE     -- 下次复习日期（DATE，非 TIMESTAMPTZ）
status           VARCHAR  -- LEARNING / REVIEWING / MASTERED
```

`ielts_review_logs` 只记录历史：`{record_id, rating, reviewed_at}`

## 触发时机

1. 用户提交评分 → `POST /api/ielts/study/review`
2. Service 调用 `SpacedRepetitionCalculator.apply(record, rating)` — 就地修改 record
3. 更新 `ielts_study_records`，写入 `ielts_review_logs` 历史
4. 今日计划生成时查询 `next_review_at <= today` 的 study_records

## SpacedRepetitionCalculator 当前实现

- **Spring Bean**（不再是纯静态工具类），构造注入 `Clock`（通过 `ClockConfig` 提供）
- 保留 `apply(record, rating)` 静态方法兼容旧调用
- `ClockConfig.java` 统一提供 `Clock.systemDefaultZone()`

## ielts_review_logs 扩展字段（2026-05-07 新增）

复习前后快照，用于算法分析：`before/after_status`、`before/after_interval_days`、`before/after_repetition_count`、`before/after_ease_factor`
