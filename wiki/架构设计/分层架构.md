# 分层架构

## 包结构

```
com.enterprise.kb
├── common/
│   ├── dto/        ApiResponse<T> · PageResponse<T>
│   └── exception/  KbException · ResourceNotFoundException · ResourceExistException
└── ielts/
    ├── config/     ClockConfig · IeltsStudyConfig
    ├── controller/ 每模块一个 Controller
    ├── dto/        请求/响应 DTO
    ├── exception/  IeltsExceptionHandler (@RestControllerAdvice)
    ├── mapper/     MyBatis Mapper 接口
    ├── model/      实体类 (对应数据库表)
    ├── service/    Service 接口
    │   └── impl/   Service 实现
    ├── study/      SpacedRepetitionCalculator (领域逻辑)
    └── typehandler/ UUIDTypeHandler
```

## 各层约定

### Controller
- `@RestController` + `@RequestMapping("/api/ielts/...")`
- 只做参数校验 (`@Valid`) 和响应包装，不含业务逻辑
- 统一返回 `ApiResponse<T>` 或 `PageResponse<T>`

### Service
- 接口 + Impl 分离，便于测试 mock
- 事务在 Service 层声明 (`@Transactional`)
- 复杂查询逻辑在 Service 中组装，不下放到 Mapper

### Mapper
- 纯 MyBatis XML，禁用注解 SQL（复杂查询可读性差）
- 文件位置：`resources/mapper/*.xml`
- 分页使用 PageHelper，在 Service 层调用 `PageHelper.startPage()`

### Model
- 字段名与数据库列名一一对应（下划线转驼峰由 MyBatis 配置处理）
- UUID 主键，`UUIDTypeHandler` 处理 PostgreSQL UUID 类型

### DTO
- 请求 DTO：`*Request.java`，带 Bean Validation 注解
- 响应 DTO：`*Response.java` 或复合字段类，从 Model 组装

## 异常处理

```
IeltsExceptionHandler (@RestControllerAdvice)
  ├── ResourceNotFoundException → 404
  ├── ResourceExistException    → 409
  ├── MethodArgumentNotValidException → 400 (校验失败)
  └── KbException               → 500 (业务异常基类)
```
