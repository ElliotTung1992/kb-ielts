# 学习计划

每日学习计划的生成、展示与执行流程。

## 核心实体

### IeltsDailyPlan
每日计划头，按日期唯一：`plan_date DATE UNIQUE`, `total_items`, `completed_items`, `generated_at`

### IeltsDailyPlanItem（2026-05-07 新增，解决计划漂移问题）
固化当日计划条目，`ielts_daily_plan_items` 表：
- `plan_id` → IeltsDailyPlan
- `content_type VARCHAR`、`content_id UUID` — 对应内容
- `record_id UUID` — 新学条目开始学习后回填（关联 study_record）
- `study_mode` — `NEW`（新学）/ `REVIEW`（复习）
- `status` — `PENDING` / `COMPLETED`
- `summary TEXT` — 内容摘要快照（防止内容删除后空白）
- `sort_order INT` — 学习顺序
- `completed_at TIMESTAMPTZ`

## 今日学习流程（2026-05-07 后稳定版）

```
GET /api/ielts/study/today-plan
  └── 查询 today 的 IeltsDailyPlan + IeltsDailyPlanItems
      ├── 不存在 → 生成计划：
      │     ├── 查询 next_review_at <= today 的 study_records（复习优先）
      │     ├── 按配额补充新内容
      │     ├── 写入 IeltsDailyPlan
      │     └── 批量写入 IeltsDailyPlanItems（固化条目，不再漂移）
      └── 已存在 → 直接读取已保存的明细条目

POST /api/ielts/study/start
  └── 写入 IeltsStudyRecord（status=LEARNING）
      └── 回填 plan_item.record_id

POST /api/ielts/study/review
  └── SpacedRepetitionCalculator.apply() → 更新 study_record
      └── 标记 plan_item.status = COMPLETED（不依赖 review log 总数）
```

## 完成进度计算（重要）

`completedItems` = plan_items 中 `status = COMPLETED` 的数量，**不再**按当天 review log 总数统计。这避免了同一条内容重复评分导致完成数虚高的问题。

## 计划生成逻辑

`IeltsStudyServiceImpl.generateTodayPlan()`：
1. 查询 `next_review_at <= today` 的 study_records（复习优先）
2. 按 `IeltsStudyConfig` 四个配额补充新内容（`dailyWords/Phrases/Grammar/Others`）
3. 写入 `IeltsDailyPlan` + `IeltsDailyPlanItems`（批量插入，附带内容摘要快照）

## 配置

`IeltsStudyConfig`（通过 `application.yml` 注入）：
- `maxNewItemsPerDay` — 每日新学上限
- `maxReviewItemsPerDay` — 每日复习上限

## 相关 API

| 路径 | 说明 |
|---|---|
| `GET /api/ielts/study/today-plan` | 获取今日计划 |
| `POST /api/ielts/study/start` | 开始学习一个条目 |
| `POST /api/ielts/study/review` | 提交复习评分 |
| `GET /api/ielts/study/plan-suggestion` | 计划建议（PlanSuggestionResponse）|
| `POST /api/ielts/study/today-plan/add` | 手动添加今日计划项 (AddTodayPlanItemRequest) |
