# 间隔复习

基于 SM-2 算法的间隔复习系统。

## 核心实体

### IeltsStudyRecord（SM-2 状态载体）
SM-2 状态存储在 `ielts_study_records`，不是 review_logs：

```
content_type      VARCHAR  -- 内容类型（WORD/PHRASE/GRAMMAR_POINT/…）
content_id        UUID     -- 内容 ID，UNIQUE(content_type, content_id)
status            VARCHAR  -- LEARNING / REVIEWING / MASTERED
ease_factor       DECIMAL  -- 难度因子（初始 2.50）— 仅 WORD/PHRASE 有效
interval_days     INT      -- 当前间隔（天）
repetition_count  INT      -- 连续成功次数
next_review_at    DATE     -- 下次复习日期
last_reviewed_at  TIMESTAMPTZ
```

### IeltsReviewLog（历史记录）
仅记录每次评分历史，不含 SM-2 参数：

```
record_id   UUID FK → ielts_study_records (CASCADE)
rating      VARCHAR  -- AGAIN / HARD / GOOD / EASY
reviewed_at TIMESTAMPTZ
```

## SM-2 算法实现

文件：`study/SpacedRepetitionCalculator.java`

**完整 SM-2（WORD / PHRASE）**：
```
AGAIN → interval=1, ef-0.20, rep=0
HARD  → interval×1.2, ef-0.15
GOOD  → 标准递增；rep≥5 且 interval≥21 时 → MASTERED
EASY  → interval×1.3, ef+0.15
```

**阶段式模式（其他内容，2026-05-07 更新）**：
```
AGAIN → LEARNING, interval=1
HARD  → LEARNING, interval=1-3 天
GOOD  → 3→7→14→30+ 天递进；rep≥4 且 interval≥30 → MASTERED
EASY  → 7→21→45+ 天递进；rep≥3 且 interval≥30 → MASTERED
```

> [!contradiction] 与 [[sources/sequence-study]] 冲突
> 原始设计文档描述简化模式为 GOOD/EASY 直接 MASTERED interval=365。
> 2026-05-07 实现已改为阶段式，以代码为准。

`apply(record, rating)` 就地修改 record，调用方再 update 到数据库。

## ielts_review_logs 扩展字段（2026-05-07）

新增复习前后快照：`before/after_status`、`before/after_interval_days`、`before/after_repetition_count`、`before/after_ease_factor`

## 错题本

`IeltsMistakeLog` 记录 quality < 3 的失败复习，支持：
- 按内容类型筛选错题
- 错题统计（`MistakeStat`）
- 专项复习错题
